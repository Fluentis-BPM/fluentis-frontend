# Variables de Entorno

Este proyecto utiliza variables de entorno para la configuración. 

## Configuración

1. Copia el archivo `.env.example` a `.env`:
   ```bash
   cp .env.example .env
   ```

2. Actualiza los valores en el archivo `.env` según tu entorno:

### Variables Disponibles

#### API Configuration
- `VITE_API_BASE_URL`: URL base de la API del backend
  - **Desarrollo local**: `http://localhost:8080`
  - **Producción**: `https://fluentis-prod-hzchhwdnavgtejdm.canadacentral-01.azurewebsites.net`

#### Azure Configuration
- `NEXT_PUBLIC_AZURE_CLIENT_ID`: Client ID de Azure AD
- `NEXT_PUBLIC_AZURE_TENANT_ID`: Tenant ID de Azure AD
- `NEXT_PUBLIC_AZURE_REDIRECT_URI`: URI de redirección para Azure AD

#### GoFile Configuration
- `VITE_GOFILE_TOKEN`: Token de autenticación para GoFile
- `VITE_GOFILE_DEFAULT_FOLDER_ID`: ID de la carpeta por defecto en GoFile

## Notas

- Las variables que comienzan con `VITE_` son expuestas al cliente y pueden ser accedidas usando `import.meta.env.VITE_*`
- El archivo `.env` está en `.gitignore` y no debe ser committeado
- Usa `.env.example` como plantilla para configurar tu entorno local
